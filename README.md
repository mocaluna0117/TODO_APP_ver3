# TODO アプリ（Flutter）

シンプルな ToDo 管理にとどまらず、**期限通知・画像添付・タグ分類・優先度・繰り返し・バックアップ**まで備えた、実用レベルのタスク管理アプリです。Flutter / Dart で開発し、iOS・Android のネイティブアプリとして動作します。

> 個人開発。要件定義からデータモデル設計、UI/UX、ローカル通知やバックアップなどの機能実装、実機デプロイまでを一人で担当しました。

## アプリ概要

「やること」「今日やること」「完了済み」「やりたいこと」の 4 つのタブでタスクを整理できる ToDo アプリです。タスクには期限・優先度・タグ・画像・繰り返し設定を付けられ、期限が近づくとローカル通知でリマインドします。データはすべて端末内に保存され、オフラインでも完結します。バックアップの書き出し / 読み込みにも対応しています。

- **対象プラットフォーム**: iOS / Android（Flutter のため Web・デスクトップへの展開も視野）
- **言語 / UI**: 日本語ローカライズ対応
- **データ保存**: 端末ローカル（外部サーバー不要・完全オフライン動作）

## 主な機能

### タスク管理

- **タスクの追加・編集・削除**（タイトル / 概要 / 期限 / 優先度 / タグ / 画像）
- **4 つのカテゴリタブ**
  - やること … 通常のタスク
  - 今日やること … 期限が今日まで（期限切れ含む）のタスクを自動抽出
  - 完了済み … 完了したタスクを集約
  - やりたいこと … いつかやりたいタスクを別枠で管理
- **カテゴリ間の移動**（「やること」⇄「やりたいこと」など）
- **完了 / 未完了の切り替え**、完了日時の自動記録
- **期限切れタスクの強調表示**

### 期限・通知

- 日付＋時刻で期限を設定
- ローカル通知によるリマインド（**期限の時間 / 10 分前 / 1 時間前 / 1 日前 / 通知しない** から選択）
- 通知タイミングを変更すると、登録済みの全タスクを自動で再スケジュール
- タイムゾーン（Asia/Tokyo）を考慮した正確なスケジューリング

### 分類・優先度

- **タグ機能**: 自由にタグを作成し、タスクへ付与・タグ単位でフィルタリング
- **優先度**: 高 / 中 / 低 / なし（★の数で視覚的に表示）
- **繰り返し設定**: なし / 毎日 / 毎週 / 毎月
- **並び順**: 期限の昇順 / 降順を切り替え

### 画像添付

- カメラ撮影・ギャラリー選択でタスクに画像を複数添付（Base64 で保存）

### バックアップ（エクスポート / インポート）

- 全タスク / 完了済みタスクを **ZIP 形式（人間が読める .txt ＋ 復元用 .json）** で書き出し、共有シートから保存・共有
- JSON / ZIP ファイルからタスクを復元（**ID 重複を検出して二重登録を防止**、未登録タグも自動取り込み）

### カスタマイズ（設定画面）

- アプリ名・各タブ名の変更
- タブの表示 / 非表示切り替え
- カラーテーマの選択（9 種類のプリセット）
- 削除確認ダイアログ・スワイプ削除の ON/OFF
- 通知タイミング・並び順・タグの管理

## アーキテクチャ・ディレクトリ構成

機能ごとにファイルを分割し、Dart の `part` / `part of` で `main.dart` を起点に 1 つのライブラリとして統合する構成を採用しています。約 4,800 行のコードを「責務の単位」で細かくファイル分割し、可読性と保守性を高めています。

```text
todo_app/
├── lib/
│   ├── main.dart                 # エントリポイント / part の集約
│   ├── app/my_app.dart           # MaterialApp・テーマ・ローカライズ設定
│   ├── app_settings.dart         # アプリ設定モデル（永続化込み）
│   ├── notification_service.dart # ローカル通知サービス（シングルトン）
│   ├── models/
│   │   └── todo_item.dart         # タスクのデータモデル
│   ├── pages/todo_home/           # ホーム画面（機能別に細分化）
│   │   ├── core/                  # 画面本体・状態・データ入出力・クエリ
│   │   ├── actions/               # 追加/削除/移動/エクスポート/インポート等の処理
│   │   ├── dialogs/               # 各種ダイアログ（追加・編集・移動・削除）
│   │   ├── form_fields/           # 日付・時刻・優先度・画像などの入力部品
│   │   └── list/                  # 一覧・カード・タグフィルタ表示
│   └── settings/                  # 設定画面（セクション/アクション/部品に分割）
├── assets/                        # アプリアイコン・起動画面画像
├── android/ ios/ web/ ...         # 各プラットフォーム固有設定
└── pubspec.yaml                   # 依存関係・アプリ設定
```

設計のポイント：

- **機能ベースのフォルダ分割**: 「画面 → core / actions / dialogs / form_fields / list」のように役割で整理し、変更箇所を素早く特定できる構成にしています。
- **`extension` による状態クラスの責務分割**: `_TodoHomePageState` の振る舞いを `_TodoHomeData`（保存・読込）、`_TodoHomeQueries`（抽出・整形）、`_TodoHomeExport` などの拡張に分けることで、1 つの巨大クラスにならないようにしています。
- **設定・通知の分離**: 設定モデル（`AppSettings`）と通知処理（`NotificationService`）を独立させ、画面ロジックから切り離しています。

## データモデル

タスク（`TodoItem`）が持つ主なフィールド：

| フィールド | 型 | 説明 |
|------------|----|----|
| `id` | int | 一意な ID（生成時のタイムスタンプ由来） |
| `title` | String | タイトル |
| `description` | String? | 概要・メモ |
| `isDone` | bool | 完了フラグ |
| `category` | String | `todo` / `done` / `future` |
| `taskTag` | String? | タグ |
| `dueDate` | DateTime? | 期限（日付＋時刻） |
| `recurrenceRule` | enum | なし / 毎日 / 毎週 / 毎月 |
| `imageBase64List` | List\<String\> | 添付画像（Base64） |
| `priority` | enum | なし / 低 / 中 / 高 |
| `completedAt` | DateTime? | 完了日時 |

`toJson` / `fromJson` で JSON との相互変換を行い、`shared_preferences` に文字列として永続化します。

## 設計上の工夫

- **後方互換性のあるデータ読み込み**: 旧フォーマット（`imageBase64` 単体や `taskCategory` など）からの読み込みにも対応し、`normalize〜` 系の関数で安全に値を正規化。アプリ更新でデータが壊れないよう配慮しています。
- **インポート時の重複防止**: 既存タスクの ID 集合と照合し、重複分はスキップ。復元件数・スキップ件数をユーザーへフィードバックします。
- **通知の整合性**: タスクの完了・削除・期限変更や、設定でのタイミング変更時に通知を確実に再スケジュール / キャンセルし、不要な通知が残らないようにしています。過去日時の通知は登録しません。
- **「今日やること」の自動集計**: カテゴリを手動で振り分けるのではなく、期限が今日まで（期限切れ含む）のタスクをクエリで動的に抽出しています。
- **UI の堅牢性**: OS の文字サイズ設定が極端でもレイアウトが崩れないよう、テキスト拡大率に上限・下限を設定。タブ名は折り返さず縮小表示するなど、表示崩れ対策を行っています。
- **エラーハンドリング**: ファイル選択・パース・共有などの失敗時に SnackBar でユーザーへ通知し、例外でアプリが落ちないようにしています。

## セットアップ・実行方法

### 前提

- Flutter SDK（Dart ^3.11）がインストール済みであること
- 実機または iOS シミュレータ / Android エミュレータ

### 依存パッケージの取得

```bash
cd todo_app
flutter pub get
```

### 実行（開発）

```bash
flutter run
```

### iPhone 実機へのデプロイ（リリースビルド）

```bash
cd todo_app
flutter run --release
```

デバイスが見つからない場合は、接続済みデバイスを確認してから ID を指定して実行します。

```bash
flutter devices
flutter run -d <デバイスID> --release
```
